Global dictation hotkeys

macOS: global hotkeys only work while run_hotkey_agent.py is running — start the stack with ./start.sh (not uvicorn alone). The sidecar registers your chords with the system via Carbon RegisterEventHotKey (not a global key tap), so Input Monitoring is usually not required for hotkeys. Accessibility may still be needed for typing the transcript into other apps (handled by the server process). Saving a hotkey here records which account the agent should use; override with env VOICE_DICTATION_AI_FRAME_USER if needed. If a chord never fires, check logs/hotkey-agent.log for RegisterEventHotKey errors or conflicts with another app using the same shortcut exclusively.

Why a separate hotkey process?

The sidecar registers only your chosen shortcuts with the OS (Carbon), runs a small CFRunLoop thread, and survives independently of the web server. That avoids coupling to uvicorn worker/reload behaviour and keeps logging (logs/hotkey-agent.log) obvious. The sidecar only talks to this app over loopback HTTP.

Preferred terms

One entry per line: product names, people, acronyms, internal codenames. The dictation LLM (when enabled) uses this list to fix spelling and choose the right form when the transcript is ambiguous. Local faster-whisper also receives a short hint (weak bias only).

For reliable names and jargon, the LLM pass is the main lever; speech models cannot guarantee a custom lexicon the way a dedicated keyword spotter would.
